What I'd recommend

pick one or ship both

Ship both as a theme toggle. Default to dark.

The Lipi renderer (WebView2 / WKWebView) already has the tooling to swap CSS variables on the fly. One `tokens.css` file with two color sets — one dark, one light — and a `data-theme` attribute on the root element. User picks their theme in Settings.

Why both, not one: the dark version is what power users want (long sessions, no eye strain). The light version is what screen-sharers and daytime users want. Apple, GitHub, JetBrains, VS Code — they all ship both. The token system makes this nearly free.

If we only ship one — Dark

Keeps Lipi's current identity. Lower risk, faster ship. Power-user favorite. Single set of tokens, single test path. The Liquid Glass effect becomes a subtle inner shine on dark panels.

If we only ship one — Light

Bigger brand shift, but the kind of move that gets the IDE noticed. The "Mac app on a wallpaper" feel is genuinely different from anything else in the IDE space right now. Riskier, more memorable.
