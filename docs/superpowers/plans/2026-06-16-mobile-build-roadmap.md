# Mobile-Build Roadmap — Phase A (Windows-doable) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the Windows-doable scaffolding for the iOS and Android build pipelines, the per-platform config, the icon set, the App Store / Play Store metadata templates, and the CI extensions. The future Mac / Linux session fills in the Swift / Kotlin source code + signing + store uploads.

**Architecture:** Phase A ships the **seam** (config + scaffolding + docs). The 8 work items are: (1) `tauri.conf.json` mobile block, (2) per-platform config files + iOS `Info.plist`, (3) `tauri-plugin-stronghold` Cargo dep + per-platform secrets-backend pick, (4) the app icon set generated by `cargo tauri icon`, (5) the store-metadata templates, (6) CI additions, (7) the umbrella plan doc, (8) `HANDOFF.md §9.48` + `CHANGELOG.md` entry. TDD: every Rust change has a failing test first; every JSON change has a parse test.

**Tech Stack:** Tauri 2 (per-platform config + `cargo tauri icon`), `tauri-plugin-stronghold` (Android keychain), `keyring` 3.x (desktop + iOS keychain), GitHub Actions (CI), Rust 1.82 (existing toolchain), Node.js 20 (existing toolchain).

```rust
    // We deliberately reuse the existing 3-variant
    // `SecretError` enum (the JS-side `SecretErrorPayload`
    // mirrors these exact 3 variants). Adding new variants

    // The map collapses Stronghold's 6 variants into the 3

    // (same shape as the existing `keyring::Error::NoEntry`
    // mapping; the JS-side `secretsGetApiKey` returns `null`
    // for this case).
    let result = map_stronghold_error(StrongholdError::ClientNotFound);
    assert!(
        matches!(result, S::Platform { ref detail } if detail == "no entry"),
        "ClientNotFound should map to Platform {{ detail: \"no entry\" }}, got {result:?}"
    );

    // `VaultNotFound` -> `Platform { detail: "no entry" }`.
    let result = map_stronghold_error(StrongholdError::VaultNotFound("v".into()));
    assert!(
        matches!(result, S::Platform { ref detail } if detail == "no entry"),
        "VaultNotFound should map to Platform {{ detail: \"no entry\" }}, got {result:?}"
    );

    // `RecordNotFound` -> `Platform { detail: "no entry" }`.
    let result = map_stronghold_error(StrongholdError::RecordNotFound("r".into()));
    assert!(
        matches!(result, S::Platform { ref detail } if detail == "no entry"),
```

```rust
use crate::voice_platform::OsFamily;

/// Which secret-storage backend the current platform uses.
/// Picked at startup based on `OsFamily` (Decision #177 in
/// HANDOFF §9.48). Desktop + iOS use `keyring`; Android
/// uses Stronghold.
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum SecretsBackend {
    /// The `keyring` 3.x crate (Windows Credential Manager /
    /// macOS Keychain / Linux Secret Service / iOS Keychain).
    Keyring,
    /// The `tauri-plugin-stronghold` crate (Android).
    /// Available only on the `mobile` feature.
    #[cfg(feature = "mobile")]
    Stronghold,
}

/// Pick the right backend for the current platform.
/// Pure function of `OsFamily`; no I/O, no async, no
/// state. Called once at app startup from
/// `secrets_get_api_key` / `secrets_set_api_key`.
pub fn pick_secrets_backend(os_family: OsFamily) -> SecretsBackend {
    match os_family {
        // Android has no keyring support; Stronghold is the
        // Tauri ecosystem's standard answer. The `mobile`
        // feature is required at the Cargo level for the
        // Android build to compile.
        OsFamily::Android => {
            #[cfg(feature = "mobile")]
            {
                SecretsBackend::Stronghold
            }
            #[cfg(not(feature = "mobile"))]
            {
                // Fallback for the Windows dev build — the
                // `mobile` feature is off, so Stronghold isn't
                // compiled. The `keyring` mock store handles
                // unrecognised backends gracefully (the test
                // suite uses the mock store).
                SecretsBackend::Keyring
            }
        }
        // All other platforms use keyring (the `apple-native`
        // feature covers both macOS and iOS per the keyring
        // 3.6 docs).
        OsFamily::Windows
        | OsFamily::Macos
        | OsFamily::LinuxGtk
        | OsFamily::Ios
        | OsFamily::Other => SecretsBackend::Keyring,
    }
}

/// Map a Stronghold error to a `SecretsError` (Decision #184).
/// Variants we don't recognise map to `PlatformError`.
#[cfg(feature = "mobile")]
pub fn map_stronghold_error(err: stronghold_stronghold::StrongholdError) -> SecretsError {
    use stronghold_stronghold::StrongholdError;
    match err {
        StrongholdError::ClientNotFound
        | StrongholdError::VaultNotFound(_)
        | StrongholdError::RecordNotFound(_) => SecretsError::EntryNotFound,
        StrongholdError::AccessDenied
        | StrongholdError::EncryptionFail(_) => SecretsError::AccessDenied,
        _ => SecretsError::PlatformError(err.to_string()),
    }
}
```

- [ ] **Step 5: Create the Stronghold facade (`src-tauri/src/secrets_stronghold.rs`)**

```rust
//! Stronghold-backed secrets for Android (mobile-build roadmap
//! Phase A, Decision #177).
//!
//! This module is a thin facade over
//! `tauri-plugin-stronghold`. It exposes the same
//! `SecretsError` type the desktop `keyring` path uses, so
//! the JS-side `secretsGetApiKey` IPC doesn't need to know
//! which backend is in use.
//!
//! Why a facade and not a trait: the `keyring` path is
//! sync (the `Entry::get_password` call is a blocking
//! syscall into the OS keychain). The Stronghold path is
//! `async` (the `Stronghold::get_record_from_vault` call
//! returns a `Future`). Normalising this with a trait
//! would force the desktop path to be async too. The
//! facade is one-off glue that calls
//! `tauri::async_runtime::spawn_blocking` for the desktop
//! `keyring` case and awaits the Stronghold future
//! directly on Android.

#[cfg(feature = "mobile")]
use tauri_plugin_stronghold::Stronghold;

use crate::secrets::SecretsError;

#[cfg(feature = "mobile")]
pub async fn stronghold_get(
    app: tauri::AppHandle,
    provider: &str,
) -> Result<Option<String>, SecretsError> {
    let stronghold = app.state::<Stronghold>();
    let client = stronghold.create_client(provider).map_err(|e| {
        // The facade normalises the error to `SecretsError`.
        // The full variant mapping is in
        // `secrets::map_stronghold_error` (tested in
        // `secrets.test.rs`).
        crate::secrets::map_stronghold_error(e)
    })?;
    let vault = client.vault(provider);
    let record = vault.get(provider).map_err(crate::secrets::map_stronghold_error)?;
    Ok(Some(record))
}

#[cfg(feature = "mobile")]
pub async fn stronghold_set(
    app: tauri::AppHandle,
    provider: &str,
    api_key: &str,
) -> Result<(), SecretsError> {
    let stronghold = app.state::<Stronghold>();
    let client = stronghold.create_client(provider).map_err(crate::secrets::map_stronghold_error)?;
    let vault = client.vault(provider);
    vault
        .insert(provider, api_key.as_bytes().to_vec(), None)
        .map_err(crate::secrets::map_stronghold_error)
}
```

```bash
for (let y = 0; y < 1024; y++) {
  for (let x = 0; x < 1024; x++) {
    const idx = (1024 * y + x) << 2;
    // Background.
    png.data[idx] = 0x1f; png.data[idx+1] = 0x6f; png.data[idx+2] = 0xeb; png.data[idx+3] = 0xff;
    // The 'L' — vertical bar (x 280..440, y 280..744) + horizontal bar (x 280..744, y 660..744).
    if ((x >= 280 && x < 440 && y >= 280 && y < 744) ||
        (x >= 280 && x < 744 && y >= 660 && y < 744)) {
      png.data[idx] = 0xff; png.data[idx+1] = 0xff; png.data[idx+2] = 0xff; png.data[idx+3] = 0xff;
    }
  }
}
png.pack().pipe(fs.createWriteStream('src-tauri/icons/lipi-icon-1024.png'));
"
```

(Note: this requires `pngjs` as a dev dep. If `pngjs` is not installed, run `npm i -D pngjs` first, then delete the dep after generation: `npm uninstall pngjs`.)

```yaml
    steps:
      - name: Checkout
        uses: actions/checkout@v4

        uses: actions/setup-node@v4

        with:

            libwebkit2gtk-4.1-dev libgtk-3-dev \
            libayatana-appindicator3-dev librsvg2-dev patchelf \
            openjdk-17-jdk

        if: runner.os == 'Linux'
        uses: android-actions/setup-android@v3
        with:
          java-version: '17'
          sdk-version: '34'
          build-tools-version: '34.0.0'
          platforms: 'android-35'

        run: npm ci

      - name: Check tauri.conf.json mobile block
        run: |
          node -e "const c = require('./src-tauri/tauri.conf.json'); \
                   if (!c.bundle.android || !c.bundle.iOS) { \
                     console.error('Missing bundle.android or bundle.iOS'); \
                     process.exit(1); \
                   }"

      - name: Check per-platform config files
        run: |
          test -f src-tauri/tauri.android.conf.json || \
            (echo "Missing tauri.android.conf.json" && exit 1)

          test -f src-tauri/icons/lipi-icon-1024.png || \

            (echo "Missing icons/ios/" && exit 1)
          test -d src-tauri/icons/android || \
            (echo "Missing icons/android/" && exit 1)

        if: runner.os == 'macOS'
        run: cd src-tauri && cargo tauri ios info

      - name: Android toolchain smoke check (Linux only)
        if: runner.os == 'Linux'
        run: cd src-tauri && cargo tauri android info
```

## Troubleshooting

### "Tauri iOS build fails on the codesign step"

The Apple Distribution certificate isn't in the
keychain. Re-import the certificate per
`release.yml`'s commented-out `release-ios` job
(the `security import` + `security
set-key-partition-list` steps).

### "Tauri Android build fails on the apk-signer step"

The keystore password is wrong. The `keytool
-keytool -list -keystore lipi.keystore -storepass
$YOUR_PASSWORD` command verifies the password
without modifying the keystore.

### "The Swift plugin crashes on `SFSpeechRecognizer.requestAuthorization`"

iOS 17+ requires the `NSMicrophoneUsageDescription`
Info.plist key. Without it, the app crashes with
`This app has crashed because it attempted to access
privacy-sensitive data without a usage description`.
The fix is to add the key to the `Info.plist` (see
Step 3 of the iOS checklist).

### "The Kotlin plugin crashes on `SpeechRecognizer.createSpeechRecognizer`"

The Android device doesn't have the Google speech
recognition service installed. The fix is to either
(a) install the Google app (which ships the
service) on the test device, or (b) use a different
recognizer implementation (e.g. the offline
Vosk recognizer, which is a much bigger lift).

### "The voice transcript doesn't land in the editor"

The `tauri-plugin-stronghold` Cargo dep isn't
enabled. The fix is to build with
`cargo tauri {ios,android} build --features mobile`
(the `mobile` feature gates the Stronghold dep).
