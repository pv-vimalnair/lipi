Lipi × Apple Liquid Glass

Apple's new Liquid Glass design system (WWDC25, iOS 26 / macOS Tahoe 26). A digital meta-material with three layers — specular highlight, shadow, refractive illumination — that reacts to movement. This is the same design system that runs on Apple TV, iPad, iPhone, and Mac in 2025. We can ship the full look in Lipi (squircle corners, specular shine, gel translucency, soft shadows) with backdrop-filter + CSS. The actual light-bending refraction needs SVG displacement maps, which is Chromium-only — so the full effect is gated behind a future WebGL/Chrome path.

The 3-layer composition

how Liquid Glass is built · per Apple WWDC25 §219
Layer 1
Highlight

Bright specular shine on the top edge — gives the gel/glass its "wet" look. Reacts to movement and light direction. In CSS: a top-half linear-gradient(rgba(255,255,255,0.45), transparent) + inset 0 1px 0 rgba(255,255,255,0.7).

Layer 2
Shadow

Soft outer drop + inner bottom shadow — creates the sense that the glass is floating above content. The shadow is light and diffuse, never harsh. In CSS: 0 12px 32px rgba(15,15,18,0.08).

Layer 3
Illumination

The translucent body itself — content beneath is refracted (bent via Snell's law, not just blurred) and tinted. We use backdrop-filter: blur(30px) saturate(180%) in our renderers. Full refraction = future WebGL.

What we can ship now (in Lipi's renderers)
  • All 3 layers of the Liquid Glass composition: specular highlight (top inner edge + ::before gradient), shadow (outer drop + bottom inner edge), illumination (translucent body + backdrop-filter).
  • Squircle-feel rounded corners (24px on panels, 12px on controls).
  • Tintable glass: --glass-bg-tinted for selected/active states (Apple's "regular" variant).
  • WebView2 (Windows) and WKWebView (macOS/iOS) both support backdrop-filter: blur() saturate() — full coverage for Lipi's targets.
What we'd need later (not for this phase)
  • Real refraction (light-bending the background) requires SVG <feDisplacementMap> as backdrop-filter: url(#filter). Chromium-only — not WebView2, not WKWebView. Future phase: WebGL refraction layer (Three.js shader with normal map).
  • Specular animation (the highlight reacting to mouse movement) needs a tiny JS hook. Trivial, can ship later.
  • Light/dark auto-adaptation via prefers-color-scheme media query on the glass tokens. Ships now, easy.
Implementation footprint
  • tokens.css: ~15 lines changed, ~13 new tokens (the glass system).
  • Component CSS: ~10-12 surface classes (panels, title bar, status bar, sidebars, modals, dropdowns) get a .glass-surface mixin or direct Liquid Glass treatment.
  • Monaco: switch built-in theme from vs-dark to vs so code matches the light glass surroundings.
  • Code blocks in chat: stay dark for code readability (matches Apple's own pattern in their apps).
